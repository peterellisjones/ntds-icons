# `ntds-icons` distribution — gallery + deploy

**Date:** 2026-06-17
**Upstream issue:** dronecom #1465, subsystem 3 (distribution).
**This document covers:** the GitHub Pages **gallery** (CLI-generated) and the
**GitHub Actions** that deploy it + publish releases. Out of scope (deferred):
npm package, `egui-ntds` integration crate, dingbat-directory submissions.

## Goal

Make the font **live and downloadable**:

1. A polished, dark-tactical **gallery** (`index.html`) generated by
   `ntds-font-gen` from the same `all_glyphs()` data as the font — so it can't
   drift. Live `@font-face` glyphs, affiliation coloring, click-to-copy
   codepoints, search + affiliation filter, and a base+heading+BDA **composition
   demo**.
2. A **Pages workflow** that regenerates artifacts and deploys the gallery on
   every push to `main`.
3. A **release workflow** that, on a `v*` tag, regenerates + drift-checks the
   artifacts and creates a GitHub Release attaching `ntds_icons.ttf` + `.woff2`.

## Non-goals (deferred)

- npm package (woff2 + CSS + JSON) — the user opted out of npm.
- `egui-ntds` integration crate — separate crate, separate effort.
- Directory submissions (DaFont / 1001fonts / Open Font Library) — manual, done
  by the user once the gallery is live.
- Republishing the crate to crates.io — not needed; the gallery is built from
  repo source via CI, and the default crate API dronecom consumes is unchanged.
  (`GlyphKey::affiliation()` is an additive, non-breaking addition; a version
  bump can wait for the next release the user chooses to cut.)

## Architecture

### 1. `GlyphKey::affiliation()` (in `layout`)

A small reusable helper so the gallery (and future consumers) can color/filter
by affiliation without re-matching the key:

```rust
impl GlyphKey {
    /// The affiliation this glyph belongs to, or `None` for the affiliation-
    /// agnostic overlays (BDA, heading).
    #[must_use]
    pub fn affiliation(self) -> Option<ShapeAffiliation> {
        match self {
            GlyphKey::Base { aff, .. }
            | GlyphKey::Geometric { aff, .. }
            | GlyphKey::Group { aff, .. }
            | GlyphKey::GroupGeometric { aff, .. }
            | GlyphKey::Unknown { aff }
            | GlyphKey::UnknownGeometric { aff } => Some(aff),
            GlyphKey::Bda(_) | GlyphKey::Heading(_) => None,
        }
    }
}
```

### 2. `build::gallery` (behind the `build` feature)

```
src/build/gallery/
  mod.rs          # render_gallery(woff2_href: &str) -> String
  template.html   # static page: dark-tactical CSS + vanilla JS + placeholders
```

- `template.html` is the maintainable source of the page — inline CSS + vanilla
  JS (no framework, no CDN), with two placeholders: `__WOFF2_HREF__` (the woff2
  filename to `@font-face`) and `__GLYPH_DATA__` (a JSON array injected by the
  generator). `include_str!`'d at compile time.
- `render_gallery(woff2_href)` builds the JSON glyph array from
  `codepoints::all_glyphs()` — one entry per glyph:
  `{ "cp": "E000", "ch": "", "name": "Air.Friend", "cat": "base",
     "class": "Air", "aff": "Friend" }` (overlays carry `"cat":"heading","deg":135`
  or `"cat":"bda","bda":"Uncertain"` and `"aff":null`) — then substitutes the
  two placeholders. `cat`/`class`/`aff`/`deg`/`bda` come from matching the
  `GlyphKey` (with `affiliation()` for the color).
- The page's JS:
  - **Grid** — all 170 glyphs, **grouped into labeled category sections**
    (Symbols · Geometric variants · Groups · Group geometric · Unknown ·
    Heading vectors · BDA) so the near-identical construction/geometric pairs
    are disambiguated by section rather than sitting side by side. Each card:
    the live glyph (affiliation-colored; overlays in the neutral accent), the
    codepoint, the name; **click-to-copy** the codepoint (e.g. `U+E000`) with a
    transient "copied" flash. The composition demo is the hero that shows the
    overlays *in use*; the grid documents that they exist.
  - **Search** — filters cards by name / codepoint substring.
  - **Affiliation toggles** — Friend / Enemy / Unknown / Neutral chips that
    show/hide cards.
  - **Composition demo** (hero) — selectors for class, affiliation, heading
    (0–355° in 5° steps), and BDA (none / Uncertain / ProbablyDestroyed). It
    composes `base_ch + heading_ch + bda_ch` (the heading/BDA are zero-advance
    combining glyphs) into one big live preview, and shows the resulting
    codepoint sequence with a copy button. Heading and BDA are mutually
    exclusive in the picker (a fresh contact shows heading; a stale one shows
    BDA — mirrors the game's `ContactPresentation`).

### 3. CLI: `ntds-font-gen` writes `index.html`

`ntds-font-gen` gains one more output: `index.html` (via
`render_gallery("ntds_icons.woff2")`), written next to the woff2 it already
emits, so Pages serves a self-contained directory (`index.html` +
`ntds_icons.woff2`). `--out-dir` still controls the destination.

### 4. Aesthetic — dark tactical

Matches the committed `specimen.png` and the font's naval-C2 identity: dark
steel-blue background (`#0d1117`-family), affiliation palette (friend `#4aa3ff`,
enemy `#ff5a5a`, unknown `#ffd23f`, neutral `#4ade80`), neutral accent for the
overlay glyphs, mono for codepoints/labels. Self-contained — no web fonts beyond
the embedded `ntds_icons.woff2` (system sans for prose).

### 5. GitHub Actions

`.github/workflows/`:

- **`pages.yml`** — on push to `main` (+ `workflow_dispatch`): checkout →
  install stable Rust → `cargo run --features build --bin ntds-font-gen --
  --out-dir dist` → upload `dist/` as a Pages artifact → deploy to GitHub Pages
  (`actions/deploy-pages`). The gallery is always current with `main`.
- **`release.yml`** — on `v*` tag push: regenerate into `dist/` → **drift
  check** (`cargo run … --out-dir assets` then `git diff --exit-code assets/` —
  fails the release if the committed `.ttf`/`.woff2` are stale vs a fresh build)
  → `softprops/action-gh-release` attaching `assets/ntds_icons.ttf` +
  `assets/ntds_icons.woff2`.

(Two files rather than one multi-trigger workflow — each has a single clear
responsibility.)

**Manual prerequisite (documented, user action):** GitHub Pages must be set to
**Source: GitHub Actions** in repo settings once, and the repo's Actions need
Pages write permission. Noted in the README + the PR.

## Tests

- `GlyphKey::affiliation()` — anchors (Base→Some, Bda/Heading→None) in `layout`
  tests.
- `render_gallery` — produces a self-contained page: contains the `@font-face`
  with `__WOFF2_HREF__` substituted, all 170 codepoints in the injected JSON,
  the composition-demo controls, and **no remaining `__…__` placeholders**
  (`build`-feature test).
- CLI smoke — `ntds-font-gen` writes a non-empty `index.html` whose head
  references `ntds_icons.woff2`.
- Workflows — validated by review + `actionlint` if available; no runtime test
  (they need GitHub's environment).

## Acceptance criteria

- [ ] `GlyphKey::affiliation()` added; tests pass.
- [ ] `build::gallery::render_gallery` generates a self-contained dark-tactical
      `index.html` from `all_glyphs()` (no drift); no placeholder leakage.
- [ ] Gallery has: live `@font-face` glyphs, affiliation coloring, click-to-copy
      codepoints, search + affiliation filter, and the base+heading+BDA
      composition demo.
- [ ] `ntds-font-gen` emits `index.html` alongside the existing artifacts.
- [ ] `pages.yml` deploys the gallery to GitHub Pages on push to `main`.
- [ ] `release.yml` drift-checks + publishes `.ttf` + `.woff2` on `v*` tags.
- [ ] README links the live gallery + documents the one-time Pages setting.
- [ ] `cargo test --features build` green; `cargo clippy --features build
      --all-targets` clean.

## Repo process

Work on the `distribution` branch; open a PR so the gallery can be eyeballed
before it merges (merging to `main` auto-deploys via `pages.yml`). The user
enables Pages (Source: GitHub Actions) once, then merges.
